1. Header & Landing Page Interface Comparison
Current Live Website
Old Interface Key Issues
  • Insecure Protocols: Loads logos and menus via HTTP links. Modern browsers flag this as "Not Secure".
  • Cluttered Navigation: Multi-tiered menus are hard to navigate on mobile devices.
  • Static Content: No interactive parent portals or direct call-to-actions.
Webtrivo Prototype
Modernized Design Features
  • Premium Aesthetics: Modern glassmorphic navigation header with subtle slide transitions.
  • Mobile Friendly: Clean centered layout designed for quick smartphone viewing.
  • Quick Access: One-tap "Student Portal" link built directly into the navbar.
2. Critical UX Failures on Current Website
  • Unsecure Mixed Content warnings: Because images are loaded over HTTP, visitors are met with insecure warnings, breaking trust.
  • Dummy/Placeholder Text: The current site's Mission and Vision segments contain standard eZoic advertisement templates and filler text ("School vision statements outline...").
  • Lacks Lead Capture: There is no way for a parent to register interest or initiate inquiries, costing the school admissions.

3. Student Admission Portal Comparison
Current Live Website
Manual Static Processes
  • No Interactive Forms: The admissions page is purely text information describing documents to submit.
  • Physical Queues Required: Parents must travel to the school office to get application forms.
  • Staff Burden: Office workers must manually write down and log student details in physical registers.
Webtrivo Prototype
Complete Digital Enrollment
  • Self-Serve Forms: Multi-step interactive online admission form with document uploads.
  • Digital Signatures: Allows parents to sign directly on their smartphone screen.
  • Instant Slip Output: Programmatically renders and downloads a double-sided PDF admission slip immediately.
4. School Administrative Desk Audit
Current Live Website
Zero Management Tools
  • Developer Lock-in: The school must contact and pay their external developer every time they need to post notices.
  • No Database Control: Cannot see past student submissions, check fee receipts, or audit files.
Webtrivo Prototype
Real-time Office Dashboard
  • Firebase Core Sync: Dynamic dashboard syncs directly to real-time firestore cloud databases.
  • CSV/Excel Sync: Roster manager allows staff to upload student lists in bulk instantly.
  • Billing Engine: Bill regular monthly fees to all portals and send SMS notices in one click.
5. Core Platform Features Compare
  • Progressive Web App (PWA): Your proposed portal is installable on Android/iOS. It works offline during connectivity drops, caching previous notifications and documents.
  • Direct Messaging Link: Embedded real-time support chat lets parents connect directly with the admissions desk from the website landing page.
